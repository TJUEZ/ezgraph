# EzGraph Design Document

## Overview

EzGraph is a creative image generation application that allows users to import text files (Markdown, plain text), enter a prompt, and generate editable high-quality diagrams using draw.io. The draw.io prompts are generated by a Large Language Model (LLM) of the user's choice.

## Core Flow

1. User imports a `.md` or `.txt` file
2. Rust backend parses the file content as context
3. User enters a prompt describing the desired diagram
4. LLM generates draw.io XML based on context + prompt
5. Frontend renders the diagram in an embedded draw.io editor for editing

## File Parser Module

**Supported Formats**: `.md` (Markdown), `.txt` (Plain Text)

**Parsing Strategy**:
- Markdown: Use `pulldown-cmark` to extract plain text content
- Plain Text: Direct read, no parsing needed
- Extensible interface for future PDF/PPT support

**Data Flow**:
```
File вҶ’ Read Bytes вҶ’ Encoding Detection (UTF-8/others) вҶ’ Parse вҶ’ Context String
```

**Error Handling**:
- File size limit: 10MB max
- Encoding failures: Try multiple encodings, fallback to UTF-8 with lossy characters
- Parse failures: Return raw content with warning flag

## LLM Adapter Module

**Supported Providers**:

| Provider  | Model Examples           | API Style           |
|-----------|--------------------------|---------------------|
| OpenAI    | gpt-4o, gpt-4-turbo      | OpenAI Chat Completions |
| Anthropic | claude-3-5-sonnet, claude-3-opus | Anthropic Messages |
| Ollama    | llama3, qwen2, codegemma | OpenAI-compatible   |
| MiniMax   | abab6.5s, abab6-chat     | MiniMax API         |
| Groq      | llama-3.1-70b, mixtral-8x7b | OpenAI-compatible |
| Custom    | Any OpenAI-compatible API | Custom base_url    |

**Core Trait**:
```rust
trait LLMProvider {
    async fn chat(&self, messages: Vec<Message>) -> Result<String, LLMError>;
    fn provider_name(&self) -> &str;
}

struct Message { role: String, content: String }
```

**Prompt Construction**:
```
System: You are a draw.io diagram generation expert. The user will provide content (context)
and a requirement. Please generate draw.io XML format diagram code. Output only XML.

User:
## Context:
{file content}

## Requirement:
{user prompt}
```

## Draw.io Rendering

**Solution**: Embed draw.io Editor via iframe

This provides full editability of generated diagrams, meeting the requirement for "editable high-quality images".

**Rust**:
- `pulldown-cmark` - Markdown parsing

- `tokio` - Async runtime
- `tauri` - Desktop framework

**Frontend**:
- `react` - UI framework
- `@tauri-apps/api` - Tauri API
- `tailwindcss` - Styling
- draw.io embed - Diagram rendering
